# 风险清单

本文件记录当前仍需持续关注的项目风险。风险不等于错误，但需要在进入下一阶段前确认是否已有规则、检查和测试覆盖。

## 高风险

| 风险编号 | 风险 | 影响 | 当前防线 |
|---|---|---|---|
| RISK-001 | AI 根据业务直觉新增 Bronze 不存在的字段 | 形成虚假 Silver 字段和错误 Gold 指标 | `check_silver_dictionary.py` |
| RISK-002 | Markdown、Excel、SQL、DuckDB schema 不一致 | Agent 和研发使用不同事实源 | `check_schema_consistency.py` |
| RISK-003 | 规则说明文档中的反例被危险模式扫描误报 | Harness 门禁不可信 | `check_dangerous_patterns.py` 已限制扫描范围 |
| RISK-004 | Silver 数据字典缺少字段来源列 | 字段血缘不可追溯 | `_gen_xlsx.py` 已生成来源列，测试已启用 |
| RISK-005 | Gold 模型尚无数据库设计入口 | Gold 规划可能绕过事实源 | `gold_database_design.md` 已创建入口 |
| RISK-009 | `standards/` 与 `database_design/`、`data_dictionary/` 重复维护规范 | 同一规则多处冲突，Agent 难以判断事实源 | `docs/standards/` 只做索引入口 |
| RISK-012 | `harness/` 复制数据库设计或字段字典 | Harness 变成第二事实源，产生规则冲突 | `harness/README.md` 明确只做工程执行入口 |

## 中风险

| 风险编号 | 风险 | 影响 | 当前防线 |
|---|---|---|---|
| RISK-006 | Text2SQL 绕过 Gold/Silver 直接查 Bronze | 问数结果口径不稳定 | `agents/text2sql/AGENTS.md` |
| RISK-007 | 枚举值未先扫描 distinct | 状态、代码、分类口径丢失 | `docs/warehouse/silver/AGENTS.md` |
| RISK-008 | 无序代理键导致重跑后主键变化 | 下游无法追溯 | `check_dangerous_patterns.py` |
| RISK-010 | 字段枚举值（状态码/标志位）缺少中文含义 | 中文用户和 Text2SQL Agent 难以理解 `PDR`、`HDR` 等缩写 | `docs/warehouse/data_dictionary/README.md` 已要求枚举值说明 |
| RISK-011 | DuckDB 文件被 DBeaver 等工具占用 | Harness 中需要读取 DuckDB 的检查无法打开数据库文件 | 运行检查前关闭占用 DuckDB 的桌面工具 |
| RISK-013 | 建表完成后未同步更新关联文档（`sql/`、`PROJECT_STATUS.md`、`docs/memory/`） | 协作者看到的文档与实际状态脱节 | `AGENTS.md` §13 变更传播规则 + §13.4 记忆写入强制要求 |

## 新增风险（Silver 构建后发现）

| 风险编号 | 风险 | 影响 | 当前防线 |
|---|---|---|---|
| RISK-014 | DuckDB SQL 方言与 MySQL/PostgreSQL 假设不一致 | 建表脚本运行失败（`INSERT OR REPLACE`、`DATE::INT`、`ROW_NUMBER` 别名引用等） | `经验复盘.md` R007-R010 + `build_silver_duckdb.py` 内置校验 + Harness 门禁 |
| RISK-015 | 大数据量下简单 MD5 产生碰撞（8,000 万行出现 86,171 次） | 主键不唯一，下游 JOIN 笛卡尔积，指标翻倍 | 每个建表函数末尾 `GROUP BY pk HAVING n>1` 校验 + `AGENTS.md` §13.6 强制作完即检 |
| RISK-016 | `TRY_CAST` 静默失败（返回 NULL 不报错） | 关键字段全表为空，下游分析结果准确但错误 | 每个 TRY_CAST 目标列建表后验证 NULL 率（经验 R008） |
| RISK-017 | `AGENTS.md` 只有规则声明，没有脚本执行 | 变更传播和经验沉淀依赖人工记忆，容易漏写 `docs/memory` | `check_memory_update.py` + `run_all_checks.py` Memory Gate |
| RISK-018 | Silver 建成后仍沿用建表前 Harness 阶段 | 实表字段漂移不能被默认门禁拦截 | `harness_targets.yml` 的 `stage: post_silver_build` + `--require-silver-tables` |
| RISK-019 | 美国日期、ISO 日期、货币字符串、千位分隔整数格式混用 | `TRY_CAST` 静默返回 NULL，日期、金额、计数类字段不可用 | `try_strptime`、金额/整数正则清洗、`check_silver_null.py` |
| RISK-020 | Gold 设计文档字段引用不存在的 Silver 字段 | Gold 建表失败或生成错误语义层，Text2SQL Agent 引用虚假字段 | `check_gold_design.py` + `tests/test_gold_design_quality.py` |
| RISK-021 | Gold 建表后实表和设计文档、中文注释不一致 | 中文问数缺少语义元数据，Agent 无法稳定解释表字段 | `check_gold_physical.py` + `tests/test_gold_build_quality.py` |
| RISK-022 | 机器翻译直接进入正式中文字段名 | 同义词混乱、缩写误译、中文口径不稳定 | `docs/warehouse/gold/AGENTS.md` + `check_gold_design.py` |
| RISK-023 | 层检查脚本只在一层落地，下一层漏建 | 每层重复"建表→发现数据有问题→门禁没拦截"的循环 | `check_gold_null.py` + `gold_sparsity_baseline.yml`；新层落地前必须创建对应的 null/sparsity 门禁 |
| RISK-024 | `human_review` 标记被当作"推迟导入"的通行证 | 外部参考数据（如官方 Excel）已可用但建表脚本硬编码 NULL，关键维表字段长期空置 | `check_gold_null.py` 全 NULL 检测 + R018 规则：human_review 仅用于数据源不存在的场景 |
| RISK-025 | 标准罚款金额被误读为实际缴款金额 | 罚单收入、已缴金额、欠款金额等指标被错误生成 | `fact_parking_violations.fine_source_status` + `gold_database_design.md` 口径说明 |
