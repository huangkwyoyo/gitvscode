# 变更复盘模板

每次发现新的问题后，按本模板写入 `经验复盘.md`，再判断是否需要更新规则、脚本和测试。

```md
## RXXX：规则标题

- 日期：YYYY-MM-DD
- 来源问题：
- 根因：
- 风险：
- 规则：
- 已落地规则文件：
- 已落地检查：
- 已落地测试：
- 待办：
```

## 判断标准

如果问题只影响一次性文案，可以只写复盘。

如果问题会影响表结构、字段、主键、指标、Join、SQL 生成或 Text2SQL 问数，必须进一步沉淀为：

1. 分层 `AGENTS.md` 规则。
2. `scripts/quality` 检查脚本。
3. `tests` 回归测试。
