# Human Review 流程定义

> 本文档定义 TianShu 项目中 `Human Review` 标记的完整生命周期管理流程。
> `Human Review` 是项目中最高频的标记之一——出现在字段含义、枚举值、指标口径、记忆条目等各个层面——但此前从未被正式定义。

---

## 1. 什么是 Human Review

`Human Review` 是一个状态标记，表示：某项信息虽然已由 Agent 生成或整理，但**其准确性、完整性或口径尚未经人工确认**。在人工确认之前，该信息不得作为正式决策的唯一依据。

### 1.1 触发条件

以下情况必须标记 `Human Review`：

| 场景 | 示例 |
|------|------|
| 字段含义不确定 | 缩写 `PDR` 的业务含义——Agent 推测可能是 "Pedestrian" 相关，但无法确认 |
| 枚举值含义不确定 | `status_code=3` 表示什么？数据字典中无说明 |
| 指标口径不确定 | "罚单金额"是标准金额还是实际缴款金额？ |
| 派生逻辑不确定 | 从多个字段计算派生字段时，公式是否正确？ |
| Join 关系不确定 | 两个表都有 `location_id`，是否确实可以 JOIN？ |
| 记忆条目待确认 | 新写入的记忆条目，尚未经 Review Agent 审查 |

### 1.2 禁止误用

以下情况**不得**使用 `Human Review` 占位：

| 误用场景 | 正确做法 |
|---------|---------|
| 数据源可用但未导入 | 直接导入数据——`human_review` 仅用于数据源不存在或无法确定的情况（见经验 R019） |
| Agent 偷懒不想查 | 先查 database_design、数据字典、DESCRIBE 结果——只有确实无法确认时才标记 |
| 审核人已知但未联系 | 先联系审核人——只有联系不上或等待超时时才标记 |

## 2. Human Review 生命周期

```
┌──────────────┐
│  标记为       │
│  Human Review │  由 Agent 或研发标记
│  创建日期：   │  必须附带：标记人、标记日期、待确认内容、建议审核人
│  标记人：     │
│  待确认内容： │
└──────┬───────┘
       │
       ▼
┌──────────────┐
│  指派审核人   │  24 小时内完成指派
│  指派日期：   │  审核人应是对此领域有专业判断的人类工程师
└──────┬───────┘
       │
       ▼
┌──────────────┐
│  审核中       │  审核人应在 7 天内给出结论
│  （等待人工） │  超时处理：自动升级至项目负责人
└──────┬───────┘
       │
       ├──→ 确认通过 → 移除 Human Review 标记，写入正式文档
       │
       ├──→ 确认有误 → 修正内容，移除 Human Review 标记
       │
       └──→ 无法确认 → 保留 Human Review，注明原因和下次复审日期
```

---

## 3. 时限规则

| 阶段 | 时限 | 超时处理 |
|------|:---:|---------|
| 标记 → 指派 | 24 小时 | 自动通知项目负责人 |
| 指派 → 审核完成 | 7 天 | 升级至项目负责人，抄送标记人 |
| 审核完成 → 文档更新 | 48 小时 | 由标记人跟进关闭 |
| 无法确认的条目 → 下次复审 | 30 天 | 再次复审或降级为"已知限制" |

---

## 4. 角色与职责

| 角色 | 职责 |
|------|------|
| **标记人**（Agent 或研发） | 识别不确定项 → 标记 Human Review → 附上上下文 → 建议审核人 |
| **审核人**（人类工程师） | 确认或修正 → 在条目上签名和日期 → 通知标记人 |
| **项目负责人** | 处理超时升级 → 确保 Human Review 不积压 → 月度审查 Human Review 状态 |
| **Review Agent** | 在审查过程中识别遗漏的 Human Review 标记 → 提醒标记人补充 |

```markdown

|--------|--------|------|------|

```

### 5.2 在字段字典中

```markdown
| 英文字段名 | 中文字段名 | 业务含义 | 治理备注 |
|-----------|-----------|---------|---------|
| `PDR` | 待确认 | Human Review：推测为 Pedestrian 相关，需人工确认（标记人：Agent，2026-06-12） | 待审核 |
```

### 5.3 在记忆条目中

```markdown
- 置信等级：L1（Human Reviewed）
- 来源人工确认：待确认——已标记 Human Review，2026-06-12，建议审核人：张三
```

## 6. 积压管理

### 6.1 月度 Human Review 审查

每个月运行以下命令查看积压情况：

```bash
# 统计所有 Human Review 标记
rg "Human Review" docs/warehouse/ contracts/ docs/memory/ --no-heading | wc -l

# 查看超过 30 天的 Human Review
# (需要专门的追踪脚本——待建设)
```

### 6.2 积压清理规则

| Human Review 积压数 | 行动 |
|:------------------:|------|
| < 5 | 正常——按流程处理 |
| 5-15 | 警告——项目负责人需在本周内推动清理 |
| > 15 | 严重——暂停新的 Agent 变更，优先清理积压 |

## 7. 与其他治理机制的接口

- **Memory 治理**：记忆条目中的 `置信等级：L1（Human Reviewed）` 表示该记忆虽经人工确认但证据链不完整 → 见 `01_memory_governance.md`
- **Schema 治理**：数据库设计文档中标记 Human Review 的字段，在 Agent 建表时不得作为正式字段 → 见 `03_schema_governance.md`
- **风险清单**：长期未处理的 Human Review 是 RISK-029（错误记忆污染）的潜在来源 → 见 `05_risk_register.md`
