# 数据仓库文档规范索引

本文件只作为规范入口，不重复维护具体规范内容。具体规则必须放在对应事实源目录，避免同一份规范在多处维护后产生冲突。

## 规范入口

| 规范主题 | 维护位置 | 说明 |
|---|---|---|
| 项目全局硬规则 | `AGENTS.md` | Agent 必须遵守的最高行为规则 |
| 数据仓库建模总规则 | `docs/warehouse/AGENTS.md` | Bronze、Silver、Gold、Meta、语义层的总规则 |
| Bronze 层规则 | `docs/warehouse/bronze/AGENTS.md` | 原始层字段、类型、追溯规则 |
| Silver 层规则 | `docs/warehouse/silver/AGENTS.md` | 标准层字段准入、来源追溯、派生字段规则 |
| Gold 层规则 | `docs/warehouse/gold/AGENTS.md` | 主题模型、事实表、维表、指标规则 |
| 数据库设计事实源 | `docs/warehouse/database_design/` | 表结构、字段、主键、类型、正式 schema 变更事实源 |
| 字段字典和枚举值 | `docs/warehouse/data_dictionary/` | 字段字典、官方数据字典索引、枚举值说明（状态码/标志位/分类代码） |
| 枚举值识别方法论 | `docs/warehouse/data_dictionary/枚举值识别方法论.md` | 如何系统性识别需要解释中文含义的枚举值字段（状态码/标志位/分类代码），事实表/维度表两套规则 |
| 建模方法论 | `docs/modeling/` | ER、维度建模、星型模型、雪花模型、ODS/DWD/DWS/ADS |
| 项目经验记忆 | `docs/memory/` | 经验复盘、风险清单、规则来源索引 |
| 变更传播规则 | `AGENTS.md` §13 | 建表/改字段/新发现后必须同步更新哪些文件，按什么顺序 |
| 质量检查脚本 | `scripts/quality/` | 一致性检查、危险模式扫描、回归测试入口 |

## 维护原则

- `standards/` 只做索引，不承载具体数据库设计。
- 字段、类型、主键、表结构的正式定义只维护在 `database_design/`。
- 字段中文名、枚举值（状态码/标志位/分类代码）的含义只维护在 `data_dictionary/`。
- 经验和风险只维护在 `docs/memory/`，成熟后再沉淀到对应规则文件和检查脚本。
- 如果本索引和具体目录冲突，以具体目录中的正式文档为准。
