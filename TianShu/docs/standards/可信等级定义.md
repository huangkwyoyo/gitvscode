# 可信等级定义（Fact Confidence Level）

> 本文档定义 TianShu 记忆系统中"可信等级"的判定标准和使用规则。
> 每条记忆条目必须在写入时声明可信等级，Agent 在读取时必须根据可信等级调整引用行为。

---

## 1. 五级可信等级

| 等级 | 英文名 | 中文名 | 含义 |
|:----:|--------|--------|------|
| **L5** | Verified Fact | 已验证事实 | 经至少两个独立来源交叉验证、并经人工审核确认的事实 |
| **L4** | Fact | 事实 | 基于硬证据确认的事实——硬证据包括：数据库 DESCRIBE 结果、SELECT DISTINCT 实际值、代码审查、数据画像结果 |
| **L3** | Assumption | 合理推设 | 基于已有规则或已知事实推导的结论，逻辑自洽但未经独立验证 |
| **L2** | Hypothesis | 假设 | 有依据的猜测，但证据不完整或存在未排除的替代解释 |
| **L1** | Human Reviewed | 人工确认 | 已经过人工审核确认，但具体证据链未保留或不可追溯——可采信但优先低于 L4/L5 |

---

## 2. 如何判定可信等级

### 2.1 判定流程

```
有数据库 DESCRIBE 或 SELECT DISTINCT 结果支撑？
    ├── 是 → 有多源交叉验证？
    │       ├── 是 + 人工确认 → L5
    │       └── 否 → L4
    └── 否 → 有代码审查或数据画像结果支撑？
            ├── 是 → L4
            └── 否 → 基于已有规则推导？
                    ├── 是 → L3
                    └── 否 → 是推测性结论？
                            ├── 有一定依据 → L2
                            └── 有人工确认但证据链不完整 → L1
```

### 2.2 常见场景对照表

| 场景 | 典型证据 | 推荐等级 |
|------|---------|:------:|
| DuckDB DESCRIBE 确认字段存在，且 database_design 文档一致 | 双源验证 | L5 |
| DuckDB DESCRIBE 确认字段存在 | 单源验证 | L4 |
| 代码审查发现某函数存在特定行为 | 单源验证 | L4 |
| 从已有规则推导出"如果 X 则 Y" | 逻辑推导 | L3 |
| "可能因为 DuckDB 的某行为导致此问题"（未验证） | 推测 | L2 |
| 人工说"这个字段是这个意思"但无书面记录 | 人工确认 | L1 |

## 3. 初始写入时的硬约束

> **初始写入时，置信等级不得高于 L3（Assumption）——除非条目中包含以下任一硬证据：**
>
> 1. 数据库 DESCRIBE 结果（注明执行日期和数据库路径）
> 2. SELECT DISTINCT 结果（注明字段名和表名）
> 3. 数据库设计文档引用（注明文档路径和对应章节）
> 4. 代码审查结论（注明审查的文件路径和行号）
>
> 基于推测、训练知识或"常识"的结论，最高只能标 L2。

此约束的目的是防止 Agent 将训练数据中的知识（可能不适用于本项目）当作事实写入记忆系统。

## 4. Agent 读取规则

Agent 读取 `经验复盘.md` 中的条目时，必须根据置信等级调整引用行为：

| 使用场景 | L5 | L4 | L3 | L2 | L1 |
|---------|:--:|:--:|:--:|:--:|:--:|
| 作为决策的唯一依据 | ✅ | ✅ | ❌ | ❌ | ⚠️ |
| 作为决策的参考之一 | ✅ | ✅ | ✅ | ✅ | ✅ |
| 写入 AGENTS.md 正式规则 | ✅ | ⚠️* | ❌ | ❌ | ❌ |
| 写入检查脚本 | ✅ | ✅ | ❌ | ❌ | ❌ |
| 写入测试用例 | ✅ | ✅ | ⚠️** | ❌ | ❌ |

> \* L4 条目写入 AGENTS.md 规则前，需额外确认一次（运行一次 DESCRIBE 或查一次 database_design）
> \*\* L3 条目写入测试仅当该测试用于"检测未来是否出现类似问题"而非"验证当前事实"

---

## 5. 置信等级变更规则

```
每次交叉验证通过 → 等级可上升一级（最高 L5）
每次发现与硬事实源冲突 → 等级下降一级
降至 L2 或以下 → 自动触发状态变更为 disputed

上升条件：
- 升 L4：至少提供 1 项硬证据（DESCRIBE / 代码审查 / 数据画像）
- 升 L5：至少 2 个独立来源交叉验证 + 1 次人工确认

下降条件：
- 降一级：与 database_design 存在已知冲突但尚未核实
- 降至 L2：经核实确认内容有误——强制标记为 disputed
```

---

## 6. 与状态的关系

| 状态 | 与置信等级的关系 |
|------|----------------|
| `proposed` | 初始写入，置信等级 ≤ L3。等待 Review Agent 或人工确认 |
| `active` | 确认后激活，置信等级基于证据强度设定（L1-L5） |
| `disputed` | 置信等级强制 ≤ L2（内容存疑），Agent 禁止引用 |
| `superseded` | 已被替代，置信等级不适用 |
| `archived` | 已归档，置信等级不适用 |
