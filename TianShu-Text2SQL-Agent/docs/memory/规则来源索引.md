## proposed → active 晋升标准

规则从 `proposed` 晋升为 `active` 至少需要满足以下条件：

1. 对应 check 已接入 fast gate warn 模式并稳定运行（连续 ≥3 次 fast gate 运行中该 check 未出现基础设施故障）。
2. 对应 pytest 至少覆盖正例和负例（即至少 1 个「应通过」和 1 个「应失败」用例）。
3. 对应 eval case 已存在并包含明确断言，或经团队评审确认不需要 eval（需在 `notes` 字段中注明原因和评审结论）。
4. 至少一次完整 fast gate 通过（所有步骤 PASS，warn checks 无 infra_fail）。
5. 人工审查确认无明显假阳性（审查记录需在 `notes` 中注明审查人和日期）。
