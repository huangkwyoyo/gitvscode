## Architecture

```
Client (127.0.0.1)
    ↓ HTTP
FastAPI (uvicorn)
    ↓
Request ID 中间件 → 安全日志
    ↓
POST /v1/ask → AgentRuntime.ask()
    ├─ 检查 is_online
    ├─ asyncio.Lock (超时 → 429)
    ├─ ThreadPoolExecutor → agent.ask() (同步)
    ├─ build_public_response()
    └─ 返回 contract v1.0
```

- **framework**: FastAPI >= 0.115
- **host**: 127.0.0.1（安全默认，不绑定 0.0.0.0）
- **concurrency model**: 单进 + 单 Agent + 异步锁 + 同步执行器
- **Agent lifecycle**: startup 创建 → shutdown close()
- **public contract**: Phase 6A v1.0（通过 build_public_response()）

## Endpoints

| endpoint | method | status | purpose |
|---|---|---|---|
| `/health/live` | GET | 200 | 进程存活（不查数据库） |
| `/health/ready` | GET | 200/503 | Agent 在线状态 + DuckDB 连接 |
| `/v1/ask` | POST | 200/422/429/500/503 | 中文问数，返回 contract v1.0 |
| `/docs` | GET | 200 | OpenAPI 文档 |

**禁止端点**（全部 404）：`/sql`, `/execute`, `/debug`, `/trace`, `/query`, `/raw`, `/admin`

## Runtime Concurrency Model

- 单进程 + 单 Text2SQLAgent
- API 层 asyncio.Lock 串行保护
- 同一时间最多一个 ask()
- 等待超时返回 429 (SERVICE_BUSY)
- Agent 内部多计划并发能力保持不变
- 同步 agent.ask() 通过 ThreadPoolExecutor 执行，不阻塞 event loop

## Agent Lifecycle

1. `startup` → `AgentRuntime.start()` → `Text2SQLAgent(...)` + `ThreadPoolExecutor`
2. 运行 → `runtime.ask(question)` → lock → agent.ask() → build_public_response()
3. `shutdown` → `AgentRuntime.close()` → `agent.close()` + executor.shutdown()

`is_online` 判定条件（全部满足）：
- context 存在
- context.offline = False
- resolver 存在
- resolver.is_connected = True

offline Agent 不得通过 `/health/ready`（返回 503），也不得接受 `/v1/ask`（返回 503）。

## Public Response Boundary

- HTTP 层只调用 `Text2SQLAgent.ask()`
- 只返回 `build_public_response()`
- 不提供 SQL/execute/debug/trace 端点
- 不接受 SQL、SQLPlan、表名、API Key、config 等客户端控制参数
- 500/503 错误使用安全 envelope，不含 traceback/SQL/数据库路径

## Known Limitations

1. **单 Agent 吞吐上限**：只支持同一时间一个 ask()，高并发场景需升级到多进程/任务队列（Phase 6C+）
2. **无真实查询超时**：DuckDB 查询无法在 Python 层面真实中断，长时间查询会一直占用 worker 线程
3. **无认证/授权**：本地 127.0.0.1 绑定依赖网络层隔离，不提供应用层认证
4. **无会话上下文**：每个请求独立，不保持对话历史
5. **无流式响应**：只有完整 JSON 响应，不支持 SSE/WebSocket
6. **offline 模式不可服务**：数据库不可用时 /health/ready 和 /v1/ask 都返回 503
