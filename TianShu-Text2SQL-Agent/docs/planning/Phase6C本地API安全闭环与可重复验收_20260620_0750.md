# Phase 6C 本地 API 安全闭环与可重复验收完成报告

## Summary

基于 Phase 6B 的只读 REST API，完成了本地安全闭环建设。开发者现在可以在一台机器上安全启动、认证调用、执行真实 DuckDB 问数、查看脱敏审计并生成可重复验收报告。

核心交付：
- 本地访问令牌认证（X-TianShu-Token）
- 进程内固定窗口请求频率限制
- 原始请求体大小限制（JSON 解析前）
- 本地脱敏 JSONL 审计
- 安全响应头（nosniff / DENY / no-referrer / no-store）
- 本地安全模式（fail-closed）
- 本地闭环 runner（自动化验收报告）
- 完整测试覆盖（90 个新测试）

## Local Closure Architecture

```
Client (127.0.0.1)
    ↓ HTTP
Body Limit 中间件（413 拒绝超大请求）
    ↓
安全响应头中间件（nosniff, DENY, no-referrer, no-store）
    ↓
Request ID 中间件（UUID 追踪）
    ↓
Auth 中间件（X-TianShu-Token → 401）
    ↓
Rate Limit 中间件（固定窗口 → 429 + Retry-After）
    ↓
POST /v1/ask → AgentRuntime.ask()
    ├─ 检查 is_online
    ├─ asyncio.Lock
    ├─ ThreadPoolExecutor → agent.ask()
    ├─ build_public_response()
    └─ 审计写入（accepted / completed / rejected）
         ↓
    harness/reports/local_api_audit/*.jsonl
```

## Local Authentication

- **header**: `X-TianShu-Token`
- **token source**: 环境变量 `TIANSHU_LOCAL_API_TOKEN`（唯一来源）
- **constant-time comparison**: `hmac.compare_digest()`
- **fail-closed**: `local_secure_mode=true` 时 token 缺失/太短 → `/health/ready` 返回 503
- **protected endpoints**: 仅 `POST /v1/ask`（健康检查无认证）
- **禁止来源**: URL query、request body、Cookie、Authorization Bearer、YAML 明文、命令行参数
- **错误处理**: 缺失/错误 token 统一返回 401，不回显令牌

## Rate Limiting

- **algorithm**: 固定滑动窗口（60 秒窗口 + burst）
- **limit**: 默认 30 requests/min + 3 burst
- **Retry-After**: 429 响应包含 Retry-After 头
- **monotonic clock**: `time.monotonic()`，不受系统时间调整影响
- **limitation**: 单进程内存限流、不按 IP、不信任代理头、不支持分布式

## Body Limit

- **max bytes**: 8192（默认）
- **rejection**: Content-Length 超限 → 413（`REQUEST_TOO_LARGE`）
- **pre-parse enforcement**: JSON/Pydantic 解析前检查，超限 body 不进入认证/限流/AgentRuntime
- **without Content-Length**: 累积 body 读取兜底检查

## Local Audit

- **file**: `harness/reports/local_api_audit/local_api_audit_<startup_timestamp>.jsonl`
- **events**: `accepted` / `completed` / `rejected`
- **allowed fields**: timestamp, request_id, event, route, http_status, response_type, duration_ms, error_code, question_length, execution_mode
- **forbidden fields**: token, question, answer, sql, generated_sql, trace, traceback, db_path, api_key, env, environment, contracts, summaries, merged_rows, chart_data
- **failure policy**: 写入失败抛出 `AuditWriteError`（不静默忽略），关键路径上不阻断请求
- **concurrency**: `asyncio.Lock` 保护单进程写入不交错
- **limitation**: 本地诊断审计，非合规级不可变审计

## API Security Headers

所有响应统一添加：
- `X-Content-Type-Options: nosniff`
- `X-Frame-Options: DENY`
- `Referrer-Policy: no-referrer`
- `Cache-Control: no-store`

本地 HTTP 不设置 HSTS。

## Known Limitations

1. **单一共享 token**：所有客户端使用同一令牌，无细粒度身份区分
2. **单进程内存限流**：重启后限流计数器丢失，不支持多进程
3. **本地 JSONL 审计不具备合规级不可篡改能力**：文件可被系统管理员修改/删除
4. **单 Agent 吞吐**：同一时刻只处理一个请求
5. **无真实查询取消**：DuckDB 长查询无法中断
6. **仅支持 127.0.0.1**：不适合公网部署
7. **非生产部署阶段**：不包含 JWT、OIDC、Redis、Docker、Nginx、Kubernetes
