# REPL 反问与拒绝示例

以下示例展示 Agent 在遇到歧义问题或危险操作时的处理方式。

## 示例 1：时间范围模糊 → 反问

```
> 最近每天有多少行程？

Agent 正在分析……

❓ 需要确认：
"最近"的时间范围不够明确。请问您指的是：
1. 最近 7 天
2. 最近 30 天
3. 最近 90 天

请选择或具体说明时间范围。
```

## 示例 2：指标不在注册表 → 反问

```
> 2026年1月司机的满意度是多少？

Agent 正在分析……

❓ 需要确认：
"满意度"指标不在当前数据仓库的注册表中。
当前支持的指标包括：行程量、事故量、万次行程事故率、行程距离、行程时长等。
请问您是否想查询其他指标？
```

## 示例 3：分组维度有歧义 → 反问

```
> 按区域统计2026年1月的行程量

Agent 正在分析……

❓ 需要确认：
"区域"可以理解为以下维度之一：
1. 行政区（Manhattan / Brooklyn / Queens / Bronx / Staten Island）
2. 交通分析区（TAZ）
3. 社区（Neighborhood）

请选择您需要的维度。
```

## 示例 4：写操作 → 拒绝

```
> 删除2026年1月之前的数据

Agent 正在分析……

🚫 拒绝回答：
系统仅支持只读查询，不支持删除、修改或插入数据等写操作。
如需进行数据管理操作，请联系数据仓库管理员。
```

## 示例 5：越权查询 → 拒绝

```
> 查询Bronze层的原始行程记录

Agent 正在分析……

🚫 拒绝回答：
不支持直接查询 Bronze 层或 Silver 层的原始/中间数据。
请使用 Gold 层的业务汇总表进行查询，或联系数据团队确认可用的数据视图。
```

## 退出 REPL

```
> /exit

再见！
```
