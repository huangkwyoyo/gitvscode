# 前端工作台计划 — TianShu DataDev Agent v3

> 文档版本：Phase 0 初稿

## 1. 目标

定义前端工作台的功能需求。工作台是工程师使用本系统的实际工作界面，不是营销页面或后台管理页面。

## 2. 定位

**不是**：
- 非营销展示页面
- 非后台管理系统
- 非数据可视化大屏
- 非移动端应用

**是**：
- 工程师日常使用的开发工作台
- 以项目书为核心的工作流界面
- 可视化展示流水线各阶段产物的只读查看器
- 人工审查和决策的操作台

## 3. 十一项功能需求

### 3.1 功能清单

| # | 功能 | 说明 | 对应流水线阶段 | Phase |
|---|------|------|---------------|-------|
| 1 | 输入/上传项目书 | 文本输入框或文件上传 | 入口 | 1 |
| 2 | 查看 RequirementIR | 展示结构化需求 | 需求分析 | 1 |
| 3 | 查看 SubIntent | 展示拆分后的子意图 | SubIntent 拆分 | 1 |
| 4 | 查看 SQLPlan | 展示 SQL 生成计划 | SQLPlan 生成 | 1 |
| 5 | 查看代码（SQL + PySpark） | 语法高亮展示代码 | 编译/开发 | 2 |
| 6 | 查看执行状态 | 实时展示每个节点的运行状态 | 执行阶段 | 2 |
| 7 | 查看交叉验证差异 | 9 维度对比结果可视化 | 交叉验证 | 2 |
| 8 | 查看差异诊断 | LLM 诊断文本展示 | 差异分析 | 2 |
| 9 | 查看返工轮次 | 展示返工历史和每次变更 | 返工 | 2 |
| 10 | 下载 Review Package | 一键下载完整审查包 | 输出 | 3 |
| 11 | 人工停止/重试 | 中断当前运行或重试 | 编排控制 | 3 |

### 3.2 功能详细说明

**F1：输入/上传项目书**
- 支持直接在文本框中输入项目书内容
- 支持上传 `.md` / `.txt` 文件
- 历史项目书可通过列表选择

**F2-F5：查看阶段性产物**
- 只读展示模式
- 结构化数据以表格/JSON 树展示
- 代码以语法高亮展示
- 支持复制内容

**F6：查看执行状态**
- 实时展示 DAG 图中每个节点的状态（等待中/运行中/已完成/失败）
- 每个节点可展开查看详细输入输出

**F7：查看交叉验证差异**
- 9 维度以绿色/红色标签直观展示通过/不通过
- 点击每个维度可展开详细差异数据
- 差异数据以表格对比展示

**F8：查看差异诊断**
- 展示 LLM 生成的诊断文本
- 支持展开/折叠

**F9：查看返工轮次**
- 显示当前是第几次返工
- 每次返工的变更对比（diff 视图）

**F10：下载 Review Package**
- 一键下载为 `.zip` 文件
- 下载前确认人工审查意见

**F11：人工停止/重试**
- 运行中可以点击"停止"中断
- 失败后可以点击"重试"重新运行当前/全部节点

## 4. Phase 0 不实现前端

Phase 0 阶段专注后端流水线核心逻辑，所有功能通过 CLI 或 API 验证。

| Phase | 前端实现范围 |
|-------|-------------|
| Phase 0 | 不实现前端，CLI 验证 |
| Phase 1 | F1-F3（项目书输入 + IR/SubIntent 查看） |
| Phase 2 | F4-F9（代码展示 + 执行状态 + 交叉验证 + 诊断 + 返工） |
| Phase 3 | F10-F11（下载 + 停止/重试）+ 完整 UI 流程 |

## 5. 界面布局（草案）

```
┌─────────────────────────────────────────────┐
│  标题栏：项目名 + 状态 + 操作按钮            │
├──────────────────┬──────────────────────────┤
│  左侧：侧边栏    │  右侧：主内容区          │
│  - 项目信息       │  - 根据当前阶段展示内容  │
│  - 流程 DAG       │  - 只读展示 + 操作入口   │
│  - 产物列表       │                          │
│  - 设置           │                          │
├──────────────────┴──────────────────────────┤
│  底部：执行状态条 + 日志                     │
└─────────────────────────────────────────────┘
```

## 6. 技术约束

- 前端框架：待定（推荐 React / Vue）
- 通信方式：REST API（后端无实时推送需求，可通过轮询实现状态更新）
- 部署方式：静态文件部署，不依赖后端渲染
- 认证方式：不涉及（用户认证由接入层处理，不在本系统范围）

## 7. 不做什么

- 不提供数据可视化编辑功能
- 不提供 SQL/Spark 代码在线编辑运行
- 不提供项目管理仪表盘
- 不提供用户权限管理界面

## 8. 测试边界

| 测试类型 | 覆盖内容 |
|----------|----------|
| API 接口 | 每个功能的后端 API 正常返回 |
| 展示正确性 | 各产物在 UI 中展示内容与后端输出一致 |
| 下载功能 | Review Package 下载内容完整 |
| 操作流程 | 项目书上传 → 查看各阶段产物 → 下载的完整流程 |
