# Phase 7：LLM 接入、安全压实与 v1.0 验收

## 目标

将 Mock LLM 替换为真实 LLM API 调用，建立完善的故障恢复和重试机制，完成安全策略的最终压实，并执行完整的 v1.0 验收。本阶段是项目从开发状态进入可交付状态的最后门槛。

- **Phase 1-6 的全部产出**：编译器、多智能体、验证器、修复循环、前端、Harness 评测框架
- **LLM API 凭证**：API Key、Endpoint URL、模型名称、速率限制配置
- **v1.0 验收检查清单**

| 产出 | 说明 |

## 模块职责

### LLM API 接入策略

- **llm_client/**：统一的 LLM API 调用封装
  - 支持多 Provider（OpenAI 兼容 API、Anthropic API 等）
  - 统一的请求/响应模型（`LLMRequest`、`LLMResponse`）
  - 请求拦截器（日志记录、Token 计数、耗时统计）
  - 响应校验器（验证响应格式是否符合预期的 Schema）
- **llm_routing/**：模型路由
  - 按任务类型分配不同模型（简单任务用小模型、复杂任务用大模型）
  - 按成本优化策略选择模型（成本敏感时优先使用廉价模型）
  - 按可用性切换 Provider（主 Provider 不可用时自动切换备用 Provider）

### 故障恢复和重试机制

- **retry_handler/**：多层重试策略
  - **网络层重试**：连接超时（3 次，指数退避，初始延迟 1s）
  - **速率限制重试**：429 响应（读取 Retry-After 头，最多 5 次）
  - **服务端错误重试**：5xx 响应（3 次，指数退避）
  - **内容过滤重试**：响应内容被安全策略拦截时（调整 Prompt 后重试，最多 2 次）
- **circuit_breaker/**：断路器模式
  - 滑动窗口统计（最近 5 分钟内错误率）
  - 阈值：错误率 > 50% 或连续失败 > 10 次 -> 断路器打开
  - 半开状态：等待 30 秒后尝试单次请求
  - 降级策略：断路器打开期间启用本地 Mock 响应
- **fallback_handler/**：降级策略
  - 第一级降级：切换备用 Provider（如 OpenAI -> Anthropic）
  - 第二级降级：切换小模型（保证基本功能可用）
  - 第三级降级：退回 SQL 确定性路径（完全脱离 LLM）

### 安全压实

- **safety_auditor/**：全链路安全审计
  - **代码生成安全**：审核 System Prompt 是否包含越狱防范、角色边界、内容过滤
  - **SQL 安全**：验证 SQL 编译器是否防止注入、只读保障、表名白名单
  - **Spark 安全**：确认 AST Validator 的 12 层规则在生产环境是否生效
  - **执行安全**：确认 DuckDB 只读模式、Spark 沙箱环境的配置正确
  - **LLM 输出安全**：检查 LLM 响应过滤器（敏感词过滤、代码注入防范）
- **safety_policy_loader/**：统一的安全策略加载器（参考 Phase 0 基线定义中的 C-2 安全修复，消除硬编码）

## 明确不做什么

- 不引入新的业务功能（仅做已有功能的"硬化"）
- 不修改 Phase 1-6 的核心架构
- 不处理生产环境的运维问题（CI/CD、监控告警、日志收集等不属于 v1.0 范围）

## 风险

| 风险 | 缓解措施 |
|------|----------|
| 真实 LLM API 响应不稳定 | 多层重试 + 断路器 + 降级策略，确保核心功能可用 |
| LLM API 成本超预期 | 成本统计模块实时监控 + 模型路由按成本优化 |
| 安全策略影响正常功能 | 安全压实阶段逐条审核，确认无过度拦截 |
| v1.0 验收发现 Blocking 问题 | 预留 2 周缓冲期用于修复 blocking 问题 |
| API 版本升级导致兼容性 | Lock 模型版本（如 gpt-4o-2026-05-01），升级前充分测试 |

## 验收标准

1. [ ] 真实 LLM API 接入成功，3 种 Provider 配置验证通过
2. [ ] 故障恢复机制全部实现并通过混沌测试（网络中断、API 限流、服务端错误）
3. [ ] 断路器在错误率超标时正确打开，恢复后自动半开
4. [ ] 降级策略三级链路可正常执行
5. [ ] 安全压实报告高风险项为 0，中风险项已全部修复
6. [ ] v1.0 验收标准清单 12 项全部通过
7. [ ] 部署配置模板完整，包含环境变量说明和启动脚本

## 测试边界

- **测试范围**：LLM API 调用、故障恢复、断路器、降级策略、安全压实、v1.0 验收
- **不测试**：Phase 1-6 的核心业务逻辑（假设已单独测试通过）
- **隔离要求**：混沌测试使用独立的 API Key 和测试账号，不干扰生产环境
- **异常测试**：API Key 无效、网络断开、连续超时、响应格式异常、安全策略拦截
